# CS-330-Comp-Graphic-and-Visualization
Portfolio repository for CS‑330 Computational Graphics and Visualization.
## Module Eight Journal Reflection

When I design software, I usually start by breaking the problem down into smaller pieces and figuring out what needs to happen first. Working on the 3D scene helped me build better planning habits, especially around organizing objects, transformations, and camera movement. I learned to think more about structure before jumping into the code. My design process for this project was mostly iterative: I built one feature at a time, tested it, and then adjusted it before moving on. This approach is something I can use in future projects because it keeps the work manageable and helps catch mistakes early.

Developing the program required me to use new strategies, especially around OpenGL, shaders, and managing multiple objects in a scene. I had to experiment a lot, and iteration played a big role in getting things to look right. Over the milestones, my coding approach became more organized and modular. I started relying more on reusable functions and cleaner logic, which made the final project easier to finish.

Computer science, especially computational graphics, gives me skills that I can use in both school and my career. Understanding how graphics and visualizations work helps me think more visually and logically at the same time. These skills can be applied to game development, simulations, UI work, or any job where visual problem‑solving matters. Overall, this project strengthened my confidence in building structured, interactive programs and gave me experience I can carry into future courses and professional projects.
